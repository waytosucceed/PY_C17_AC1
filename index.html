BASIC OF FUNCTIONS

Before we jump into today’s topic, let’s take a moment to refresh our memory on the key concepts we’ve learned about functions, parameters, and arguments.

Function:
It’s a set of instructions that performs a specific task. You can call a function multiple times to avoid writing the same code over and over again.

For example, color_my_drawing() is a function.

Parameters:
Parameters are the values we define while creating a function. Think of parameters as placeholders that the function uses to perform its task.

In the example, color is the parameter.

Arguments:
Arguments are the actual values that we pass into the function when we call it.

For example, 'red' is the argument we pass into the color_my_drawing() function.

Now that we’ve refreshed our understanding of functions, parameters, and arguments, let’s dive into default parameters.

ARBITRARY ARGUMENTS

This brings us to arbitrary arguments.

When you're unsure about the number of arguments your function might receive, you can handle that gracefully by using an asterisk (*) before the parameter name. This allows the function to accept any number of arguments!

Let’s look at a simple example where we don’t know how many colors we’ll be passing to the function. We use the * symbol to allow the function to accept any number of arguments.




In this case, the function print_colors() accepts any number of arguments, which are all stored as a tuple in the colors parameter. You can pass as many colors as you want, and the function will print them all!

You can use the len() function to find out how many arguments were passed. Since the arguments are stored as a tuple, you can also access the values using indexing.

In Python documentation, you will find *args as a short form for arbitrary arguments.

RULES FOR USING MULTIPLE TYPES OF ARGUMENTS

  • Positional Parameters (param1, param2, ...)
  • Default Parameters (param=default_value)
  • Arbitrary Arguments (*args)
  • Arbitrary Keyword Arguments (**kwargs)
